# UI_DECISIONS.md

## 1. UI Direction
The current design references are preference inputs, not locked implementation specs.

## 2. Core UI decisions
1. English-only maintained UI
2. Mobile-friendly first, but Web remains primary implementation target first
3. Fast food entry is the most important interaction
4. Visual simplicity beats decorative complexity
5. One clear design direction should be frozen before full implementation

## 3. What to keep from current design exploration
- Minimal healthy aesthetic
- Cleaner card hierarchy
- Fast access to food search
- Saved meal access should support quick repeat logging
- Unified Web/App visual direction

## 4. What is flexible
- Component styling details
- Exact card arrangement
- Microcopy details
- Visual spacing and icon choices
- Final page-by-page layout adjustments

## 5. Required screens for MVP
- Today / Dashboard
- Food search and add
- Daily meal log
- Targets/settings
- Saved meals/templates (if included in MVP)

## 6. Interaction priorities
1. Add food with minimum taps
2. See daily totals quickly
3. Edit/delete quickly
4. Keep mobile usage comfortable
5. Avoid clutter

## 7. Language standard
All maintained UI copy should be English.
Legacy Chinese labels should not be extended into the new product.

## 8. Design freeze rule
Before implementation starts, select one preferred direction and treat it as the baseline. Further design changes should be incremental, not structural.

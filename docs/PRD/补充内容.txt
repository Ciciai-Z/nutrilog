一些可以借鉴的开发规范：
在 HTML 和 JavaScript 编程中，核心原则通常围绕结构分离、可维护性和用户体验展开。
1. 关注点分离 (Separation of Concerns)
这是前端开发最基本的原则，要求将结构、表现和行为拆分开： 
HTML (结构)：仅负责页面的语义和内容。
CSS (表现)：负责样式和布局，避免在 HTML 中使用内联样式。
JavaScript (行为)：负责逻辑和交互。应避免在 HTML 标签中使用 onclick 等内联脚本，改用 DOM 事件监听器。 
2. 语义化 (Semantic HTML)
使用正确的标签：使用 <header>、<article>、<nav> 等具有语义的标签，而不是到处使用 <div>。
意义：提高代码的可读性，增强 SEO（搜索引擎优化），并让辅助设备（如屏幕阅读器）能更好地理解网页结构。 
3. JavaScript 编码原则
在编写脚本时，遵循以下原则可以显著提升代码质量： 
单一职责原则 (SRP)：一个函数或模块只做一件事，这样更易于测试和复用。
避免全局变量：尽量使用 let、const 或模块化开发，防止全局作用域污染和变量冲突。
DRY 原则 (Don't Repeat Yourself)：提取重复的代码逻辑到可复用的函数或组件中。
KISS 原则 (Keep It Simple, Stupid)：保持逻辑简单直观，不要过度设计。
SOLID 原则：在复杂的项目中应用面向对象设计原则，如开闭原则（对扩展开放，对修改关闭）。 

4. 渐进增强与优雅降级
渐进增强 (Progressive Enhancement)：首先确保网页在最基础的浏览器上能正常显示内容，然后再为高级浏览器添加复杂的交互。
优雅降级 (Graceful Degradation)：先构建完整的功能，再针对旧版本浏览器提供备用方案。 

5. 性能与安全
命名规范：使用有意义的变量名（如小驼峰式命名法 myVariable），提高代码可读性。
异步加载：使用 async 或 defer 属性加载脚本，避免阻塞页面渲染。
安全实践：在处理用户输入时，始终进行过滤和转义，以防止跨站脚本攻击 (XSS)。
